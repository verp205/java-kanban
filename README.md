# Java Kanban Board 📋 — Менеджер задач

**Java Kanban** — это консольное приложение для управления задачами, реализованное в рамках обучения принципам объектно-ориентированного проектирования. Проект демонстрирует умение работать со сложными структурами данных, сериализацией и архитектурными паттернами.

---

### 🛠️ Технологический стек

* **Core:** Java 11 (Collections Framework, Generics).
* **Data Management:** File I/O (хранение данных в CSV), работа с HTTP-запросами.
* **Architecture:** In-memory и File-backed менеджеры, паттерн "Обработчик" (Managers).
* **Testing:** JUnit 5 (полное покрытие логики менеджеров и истории просмотров).
* **Infrastructure:** Работа с кастомным HTTP-сервером для API доступа.

---

### 🚀 Ключевые возможности

* **Типы задач:** Поддержка обычных задач, эпиков и подзадач (Subtasks) с автоматическим расчетом статуса эпика.
* **История просмотров:** Реализация кастомного связного списка (Linked List) для хранения истории последних просмотров без дубликатов за **O(1)**.
* **Приоритезация:** Автоматическая сортировка задач по времени начала (StartTime) с проверкой пересечений интервалов.
* **Хранение данных:** * `InMemoryTaskManager` — для быстрой работы в оперативной памяти.
    * `FileBackedTaskManager` — для сохранения состояния в файл и восстановления после перезапуска.
* **HTTP API:** Доступ к управлению задачами через REST-подобные запросы.

---

### 🏗️ Архитектура

Проект построен по принципам слабой связанности (Loose Coupling):
1. **Model:** Четкая иерархия классов (Task -> Epic -> Subtask).
2. **Service:** Интерфейс `TaskManager` и его реализации, разделяющие логику хранения и управления.
3. **Utility:** Класс `Managers` для централизованного создания объектов (Dependency Injection вручную).
4. **History:** Отдельный менеджер для управления историей просмотров.

---

### 🧪 Тестирование

Особое внимание уделено качеству кода:
- Unit-тесты для каждого типа менеджера.
- Тестирование граничных условий (пустые списки, неверные ID, пересечение временных интервалов).
- Тестирование восстановления состояния из файла.
